# 浏览器发布运行手册(真实公众号)

> 适用:**未认证个人订阅号**(也适用认证订阅号/服务号,按实际后台微调选择器)。
> 教学演示用测试/模拟号(account_type=3)走 MockBrowserChannel,**不需要**本手册。
> 本手册面向"用自有真实账号真跑一次发布"。

## 0. 合规前置(务必先读)

浏览器自动化操作微信公众号后台属灰色地带(微信有反自动化机制、《运营规范》有约束)。**只用你自己的账号、低频操作、自担风险**;禁止代他人操作、禁止规模化滥用、禁止绕过任何验证。

## 1. 安装 Playwright

```bash
cd backend
.venv/Scripts/python -m pip install -r requirements-dev.txt
.venv/Scripts/python -m playwright install chromium      # 下载浏览器内核(~150MB)
```

## 2. 在系统里把你的公众号建成"真实号"

- 后台「设置与开发 → 基本配置」能看到你的 **AppID**(形如 `wx1234...`)。
- 在本系统「公众号台账 → 新建公众号」:
  - **账号类型选「订阅号」(account_type=1)** —— 只要不是"测试/模拟号(=3)",网关就会路由到真实 `BrowserChannel`。
  - AppID 填你的真实 AppID;**AppSecret 可随便填**(浏览器发布不用 secret,用的是登录 cookies)。

## 3. 扫码捕获登录态(一次,长期复用)

```bash
.venv/Scripts/python scripts/wx_login.py wx你的AppID
```
弹出浏览器 → **用该公众号管理员微信扫码登录** → 登录态自动存到 `secrets/wx_state/wx你的AppID.json`(已 gitignore,等同凭据勿外传)。过期后重跑本命令即可。

## 4. 安全试跑(DRY_RUN,强烈建议先做)

先"只建草稿、不真发",核对自动化把内容填对了、并借截图校准选择器:

在 `backend/.env` 设:
```
WX_HEADLESS=false          # 看得见浏览器操作
WX_PUBLISH_DRY_RUN=true    # 到"保存草稿"为止,不点发表
```
然后走一遍:内容中心编辑一篇图文 → 提审 → 审核通过 → 发布任务(或首页「手动执行流水线一次」)。
- 观察浏览器逐步:登录→新建图文→填标题正文→保存;
- 去公众号后台「草稿箱」确认草稿建对了;
- 对照 `media/wx_debug/*.png` 截图。**哪一步没点对/没填对 → 改 `backend/app/modules/wx_gateway/browser_channel.py` 里的 `WX_SELECTORS`**(每项可写多个候选,逗号分隔)。

## 5. 真跑「发表」(发布到主页,不推送粉丝,可全自动)

试跑 OK 后,`.env` 改回:
```
WX_PUBLISH_DRY_RUN=false
WX_HEADLESS=true           # 或 false 继续观察
```
新建发布任务(默认 `publish_type=1` 发表)→ 系统自动:登录→建稿→填充→保存→发表→回读成文链接。发表**不推送粉丝、无次数限制**。

## 6. 「群发」(推送粉丝,可选,半自动)

创建任务时 `publish_type=2`(群发)。系统点"群发"后**需要你用管理员微信扫码/短信确认**(脚本会等待,最长 `WX_MASS_CONFIRM_WAIT_S` 秒)。
- ⚠️ **订阅号群发 1 次/天**,超了会失败;
- 群发无法完全无人值守(确认环节是微信强制的人工步骤)。

## 7. 选择器校准(必然要做的迭代)

微信后台页面结构随版本变化,`WX_SELECTORS` 里的选择器**一定需要按你的实际页面调整**。流程:
1. `WX_PUBLISH_DRY_RUN=true` + `WX_HEADLESS=false` 跑一次;
2. 卡在哪一步(报错 + `media/wx_debug/` 截图);
3. 用浏览器 F12 找到该元素的真实选择器,填进 `WX_SELECTORS` 对应项;
4. 重跑,直到走通"保存草稿";
5. 再关 DRY_RUN 真发。

**把失败步骤的报错 + 截图发给我,我帮你改对应选择器。** 这是接入真实后台的正常过程,不是一次成型。

## 关键配置项(backend/.env)

| 变量 | 默认 | 说明 |
|---|---|---|
| `WX_STORAGE_STATE_DIR` | `secrets/wx_state` | 登录态目录(每号一个 `<app_id>.json`) |
| `WX_HEADLESS` | `true` | 无头;调试设 false 观察 |
| `WX_PUBLISH_DRY_RUN` | `false` | true=只建草稿不发表(安全试跑) |
| `WX_DEBUG_SHOT_DIR` | `media/wx_debug` | 每步/失败截图 |
| `WX_STEP_TIMEOUT_MS` | `20000` | 单步等待选择器超时 |
| `WX_MASS_CONFIRM_WAIT_S` | `180` | 群发等待管理员确认最长秒数 |

## 已知局限(教学首版)

- **正文图片上传**是最需按实际编辑器校准的一步:未认证号站外图片要经编辑器上传按钮传到微信图库。首版对纯文本/已在图库的图较稳;带大量站外图的正文需迭代该步。建议第一篇真跑用**纯文本或少图**内容。
- 多图文组(一次≤8篇)首版按单篇处理,组发布后续扩展。
- `takedown`(一键下架)真实实现需按你后台"已发表列表"页面接入(当前真实号下架为占位)。
